# Техническое задание на проект "Консольная утилита dir_humanize"

## 1. Общая информация
**humanize** — библиотека Python для преобразования «машинных» данных в человекочитаемый вид: числа с разрядами и словами (1.2 million), относительное и абсолютное время («3 минуты назад», «вчера»), размеры файлов (1.0 MB, 976.6 KiB), дроби и научная нотация. Поддерживает множество локалей, в том числе русскую.

**Цель проекта:** Создать консольное приложение для наглядной демонстрации возможностей библиотеки `humanize` путем разработки полезного инструмента для работы с файловой системой.

## 2. Описание задачи (что нужно сделать)
Необходимо разработать консольную утилиту `dir_humanize`, которая выводит информацию о содержимом директории в формате, понятном человеку. Вместо "сырых" данных (размер в байтах, timestamp) утилита должна представлять их в удобочитаемом виде с использованием библиотеки `humanize`. Проект должен быть выложен в Git с полной историей разработки и подробной документацией.

## 3. Функциональные требования (что система должна уметь)
- **Анализ директории:** Утилита должна принимать путь к директории в качестве аргумента. Если путь не указан, должна анализироваться текущая директория.
- **Вывод информации о файлах:** Для каждого файла в директории (без рекурсивного обхода вложенных папок) необходимо выводить:
    - Имя файла.
    - Размер в человекочитаемом формате (например, "12.5 KB", "1.0 MB") с использованием **`humanize.naturalsize`**.
    - Дату и время последнего изменения в относительном (например, "3 минуты назад", "вчера") с использованием **`humanize.naturaltime`** и **`humanize.naturalday`**.
- **Сводка по директории:** В конце вывода должна отображаться итоговая статистика:
    - Общее количество файлов с разделителями разрядов (например, "1,234" или "1 234") с использованием **`humanize.intcomma`**.
    - Общее количество папок с разделителями разрядов.
    - Суммарный размер всех файлов в человекочитаемом формате с использованием **`humanize.naturalsize`**.
- **Локализация:** Утилита должна поддерживать переключение языка вывода через аргумент `--locale`. Должна быть реализована поддержка как минимум двух локалей: английской (`en_US`) и русской (`ru_RU`) с использованием **`humanize.i18n.activate()`**.
- **Бинарные/десятичные единицы:** Утилита должна поддерживать переключение между десятичными (KB, MB) и бинарными (KiB, MiB) единицами измерения размера через флаг `--binary`, используя параметр `binary` функции **`humanize.naturalsize`**.
- **Обработка ошибок:** Программа должна корректно обрабатывать ситуации, когда указанная директория не существует, нет прав на её чтение, или запрошенная локаль не найдена, выводя понятное сообщение об ошибке.

## 4. Нефункциональные требования (ограничения)
- **Язык программирования:** Python 3.8+
- **Используемые библиотеки:** 
    - `humanize` (основная библиотека для форматирования)
    - Стандартная библиотека Python (`os`, `datetime`, `argparse`, `sys`)
- **Интерфейс:** Консольный (CLI)
- **Архитектура:** Модульная (функции разделены по ответственности: парсинг аргументов, сканирование, форматирование).
- **Код должен соответствовать PEP 8.**
- **Проект должен запускаться на Windows, Linux и macOS.**

## 5. Требования к интерфейсу
- Интерфейс реализуется через командную строку с использованием модуля `argparse`.
- **Аргументы:**
    - `[directory]` — необязательный позиционный аргумент, путь к директории (по умолчанию текущая).
    - `--locale LOCALE` или `-l LOCALE` — необязательный аргумент для выбора локали.
    - `--binary` — необязательный флаг для вывода размеров в бинарных единицах (KiB, MiB).
- **Формат вывода:**
    - Заголовок с указанием анализируемой директории.
    - Список папок с префиксом `[DIR]`.
    - Список файлов, каждый с новой строки, с выравниванием колонок. Формат: `[РАЗМЕР] [ИМЯ] [ОТНОСИТЕЛЬНОЕ ВРЕМЯ]`.
    - Пустая строка для разделения.
    - Блок со сводкой.

## 6. Входные и выходные данные
- **Входные данные:**
    - Аргументы командной строки (путь к директории, локаль, флаг binary).
    - Содержимое файловой системы по указанному пути.
- **Выходные данные:**
    - Текстовый отчет в консоль, содержащий список файлов/папок и итоговую статистику, отформатированную с помощью `humanize`.
    - Сообщения об ошибках в `stderr`.

## 7. Критерии приёмки (когда работа считается выполненной)
Работа считается выполненной, если:
1. Реализованы все функциональные требования из пункта 3.
2. Программа запускается командой `python dir_humanize.py [directory] [--locale LOCALE] [--binary]` без ошибок.
3. Вывод программы корректен, информативен и соответствует требованиям к интерфейсу.
4. Библиотека `humanize` используется для всех типов форматирования:
   - `humanize.naturalsize` для размеров.
   - `humanize.naturaltime` и `humanize.naturalday` для времени.
   - `humanize.intcomma` для чисел в сводке.
   - `humanize.i18n.activate/deactivate` для смены локали.
5. Локализация работает: при указании `--locale ru_RU` время и числа выводятся на русском языке.
6. Флаг `--binary` корректно переключает единицы измерения размеров.
7. Программа обрабатывает ошибки доступа и несуществующие пути.
8. Код организован логично с понятными именами функций и переменных.
9. Проект загружен в публичный Git-репозиторий.
10. В репозитории присутствует файл `README.md` с описанием проекта.

## 8. Дополнительные условия
